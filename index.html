Soal 1: Skrip Eksekusi (PowerShell)

Untuk menjaga integritas barang bukti dan mempercepat proses akuisisi data, kami menerapkan otomatisasi hashing menggunakan Windows PowerShell. Skrip ini secara otomatis menelusuri direktori barang bukti, menghitung nilai tanda tangan digital SHA-256 dari setiap berkas, dan mengekspor hasilnya langsung ke format laporan terstandarisasi CSV.

powershell
# Navigasi ke folder barang bukti
cd "C:\BarangBukti_Mahasiswa"

# Hitung hash SHA-256 semua file sekaligus
Get-ChildItem "C:\BarangBukti_Mahasiswa" -File | Get-FileHash -Algorithm SHA256

# Ekspor hasil ke file CSV
Get-ChildItem "C:\BarangBukti_Mahasiswa" -File | Get-FileHash -Algorithm SHA256 | Export-Csv -Path "hash_report.csv" -NoTypeInformation

Soal 3: 5 Keunggulan Otomatisasi

Mengapa investigasi digital forensik harus meninggalkan metode penanganan bukti konvensional? Berikut adalah 5 aspek keunggulan penerapan otomatisasi PowerShell dalam rantai penanganan bukti (chain of custody):

01 / Efisiensi

Efisiensi Waktu

Kerja < 5 Detik

Proses komputasi berjalan secara simultan pada level kernel. Skrip sanggup memangkas waktu kerja berjam-jam menjadi hitungan detik.

02 / Presisi

Minimalisasi Human Error

Nol Risiko Typo

Menghilangkan manipulasi manual, penyalinan manual, atau kesalahan input karakter hash yang dapat merusak legalitas bukti di persidangan.

03 / Standarisasi

Dokumentasi Konsisten

Ekspor CSV Otomatis

Setiap hasil kalkulasi secara terstruktur diarsip ke berkas CSV yang siap dikomparasi dan dilampirkan langsung pada berkas laporan resmi.

04 / Auditabilitas

Reprodusibilitas Mutlak

Bisa Diulang Kapan Saja

Investigator lain atau auditor pengadilan dapat menjalankan ulang skrip yang sama untuk memvalidasi ulang keaslian barang bukti.

05 / Performa

Skalabilitas Maksimal

Ribuan Bukti Sekaligus

Tanpa limitasi jumlah file bukti. Skrip sanggup melakukan rekursif mendalam menembus ribuan sub-direktori dengan stabil.

Soal 4: Komparasi Manual vs PowerShell

Untuk mempertegas superioritas metode otomatisasi, tabel komparasi berikut membandingkan aspek-aspek utama proses akuisisi integritas data antara teknik konvensional dan eksekusi skrip PowerShell:

Aspek Pengujian Metode Manual (GUI / Online) PowerShell (Otomatisasi) Pemenang Mutlak
Kecepatan 2 - 3 Jam (Memilih berkas satu per satu, mengkalkulasi, lalu menyalin teks secara mandiri).
< 5 Detik Mengekstraksi seluruh direktori instan dalam satu kali eksekusi perintah shell.

Terstandarisasi Otomatis Ekspor instan ke format file CSV universal, meminimalisir pengerjaan ulang berkas laporan.

Soal Bonus: Otomatisasi Verifikasi & Deteksi Integritas

Sebagai langkah preventif terhadap sabotase barang bukti digital, skrip bonus berikut dirancang untuk memuat kembali berkas laporan hash_report.csv, menghitung ulang nilai hash dari file barang bukti secara real-time, lalu membandingkannya secara otomatis untuk mendeteksi perubahan sekecil apa pun pada data.

powershell - Verifikator Hash
# Membaca laporan hash CSV yang ada
$recordedHashes = Import-Csv -Path "hash_report.csv"

# Melakukan verifikasi untuk setiap file
foreach ($row in $recordedHashes) {
    $currentHash = (Get-FileHash -Path $row.Path -Algorithm SHA256).Hash
    
    if ($currentHash -eq $row.Hash) {
        Write-Host "MATCH: Berkas [$(Split-Path $row.Path -Leaf)] utuh dan tidak dimodifikasi." -ForegroundColor Green
    } else {
        Write-Warning "MISMATCH: Berkas [$(Split-Path $row.Path -Leaf)] telah termodifikasi atau rusak!"
    }
}
